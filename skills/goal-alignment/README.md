# goal-alignment

**The loop, and the heart of the repo.** Layers on `pair` and `teach`.

One idea: do all the thinking *before* the agent starts looping. Resolve every open decision, prove every prerequisite live, and co-write a plain-english brief (Problem, Implementation, Verification) you can read aloud. Then hand the agent a self-correcting loop whose only "done" criteria are checks that actually land in the transcript, judged each turn by an independent evaluator that can be fooled by a confident claim, so the brief forces user-visible proof and a fresh-context critic. Preparation is the product; the loop is the cheap part.

See [SKILL.md](SKILL.md) and the [piv-template.md](piv-template.md) you fill in per round.
